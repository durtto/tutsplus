Usando Bower PAra Aprimorar o Desenvolvimento no WordPress

O Bower é muito bom para equipes, porque ele, basicamente, auto-documenta suas dependências front-end. Por conta disso, é muito fácil atualizá-las quando seu projeto precisa mudar. É ótimo para o processo de construção do seu projeto, tanto localmente, quanto em um servidor. Você também pode usar o Bower para buscar seus próprios repositórios privados, permitindo o reuso entre vários projetos.

No post anterior, falei sobre o que o Bower é e alguns de seus recursos. Nesse post, falarei sobre os benefícios em usar o Bower e como aprimorar seu processo de desenvolvimento no WordPress.

É Auto-Documentável

Como já falei, Bower é muito bom para equipes, pois ele auto-documenta suas dependências front-end. Se você leu meu post anterior sobre Grunt, o processo é muito parecido. Quando você quer instalar um pacote através do Bower, você digita bower install package-name, e, adicionando --save ao final do comando, fará com que o pacote seja adicionado à lista de dependências do projeto, no arquivo bower.json.

Digamos que você queira incluir o Bootstrap no seu tema do WordPress e quer usar os arquivos .scss durante o desenvolvimento. Você, provavelmente, quererá usar o projeto bootstrap-sass-official e instalá-lo através do Bower. 

Eis como você deve proceder:

  1. Abra o aplicativo de linha de comando de sua preferência
  2. Navegue até a pasta raiz do seu projeto
  3. Digite bower install bootstrap-sass-official --save e aperte "Return" (ou "Enter")

Se tudo foi instalado corretamente, você deveria ver duas coisas: um novo diretório, chamado bower_components, e a linha boostrap-sass-official adicionada ao seu arquivo bower.json.

Você fará isso para cada pacote que quiser usar em seu projeto. Uma vez feito isso, qualquer outro desenvolvedor do projeto ou colega de equipe poderá instalar todos os pacotes necessários. Tudo o que eles precisarão fazer é digitar bower install e todos os pacotes serão baixados e instalados na versão local do projeto de cada um deles.

Claro, essa técnica não está limitada, somente, a frameworks front-end como o Bootstrap. Há um repositório inteiro e filtrável de pacotes que estão disponíveis. Isso pode tornar a administração de dependências do WordPress muito mais fácil.

Atualizando Dependências

Novamente, não é incomum que os pacotes sejam atualizados, certo? Esse é um outro ponto em que o Bower é bem útil.

Agora que você tem todas as dependências do seu projeto documentadas e instaladas, é muito fácil atualizar esses pacotes quando atualizações para eles estiverem disponíveis. Tudo que precisa fazer é digitar bower update e o Bower lerá todas as dependências no seu arquivo bower.json e verificará se há atualizações para baixar.

O modo que o Bower faz para descobrir se há alguma atualização disponível é olhando a versão rotulada da dependência no arquivo bower.json e comparar com a versão rotulada mais recente do projeto no repositório do mesmo, lá no GitHub. Você pode descobrir quais versões rotuladas um projeto tem, olhando a página de lançamentos, como a página de lançamentos do Boostrap Sass.

Para entender mais sobre o significado das versões rotuladas, você deveria acessar a página semantic versioning. É isso o que a maioria dos projetos registrados no Bower usam em seus lançamentos, então, é interessante entender o que cada número significa.

Processo de Construção

Os pacotes que são instalados através do Bower são necessários, somente, para a construção ou compilação do projeto. Você não precisa adiciona-los ao seu repositório. O bower_components é um diretório temporário, então, sugiro que você adicione um arquivo .gitignore ao repositório do seu projeto, para que esse diretório não seja adicionado.

Se você tiver um servidor de construção ou de integração continua para seu projeto, você quererá garantir que o comando bower install seja executado bem no começo da construção. Uma vez que todas suas tarefas forem executadas, você pode apagar a pasta bower_components ou garantir a execução do comando bower update em cada outra construção.

Se você usa o Grunt para a construção de suas tarefas, há um plugin chamado grunt-bower-task que pode executar o comando bower install por você.

Repositório Privados

Você pode usar o Bower para enxergar seus respositórios privados também. Isso é ótimo para o reuso de código entre os projetos. Digamos que você tenha um modelo ou uma framework customizada que você usa em cada um dos seus projetos. Você poderia salvá-los em seus próprios repositórios no GitHub e, então, buscá-los para outros projetos através do Bower.

Você precisará ter certeza de registrá-lo na Lista pública do Bower, para poder encontrá-lo. Não se preocupe, se alguém não tiver acesso ao repositório, ela não poderá baixá-lo.
