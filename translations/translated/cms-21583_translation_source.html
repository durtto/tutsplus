Parte Um: Uma Introdução às Ações para WordPress

Começamos com uma simples introdução ao mundo das ações no WordPress. Uma vez que o sistema inteiro é bem simples, não foi difícil apresentá-lo. Aprendemos as sete funções essenciais abaixo:

  • add_action() que nos permite atrelar nossas funções a ganchos de ações existentes;
  • remove_action() que remove uma função de um gancho de ação existente;
  • remove_all_actions() que remove todas as funções de um gancho de ação;
  • do_action() que nos permite criar novos ganchos de ação;
  • do_action_ref_array() que nos permite criar novos ganchos de ação, usando um vetor de argumentos;
  • did_action() que conta o número de vez que uma ação foi executada;
  • e a has_action() que verifica se há alguma coisa atrelada a uma ação.

50 Ótimos Exemplos de 50 Ótimas Ações

No decorrer dos cinco artigos posteriores à introdução, fomos apresentados a 10 ações existentes no código do WordPress, com exemplos para cada um deles. 

Lembremo-nos para que serviam cada um deles:

Parte Dois: 50 Ações Para WordPress – 50 Exemplos (1 a 10)

  1. init lida com a inicialização do WordPress – logo antes dos cabeçalhos serem enviados.
  2. send_headers envia os cabeçalhos e nos permite adicionar cabeçalhos extras.
  3. after_switch_theme é executada logo após o usuário trocar de tema.
  4. manage_posts_custom_column nos permite adicionar colunas adicionais à página "Todos os Posts" no painel administrativo.
  5. admin_head Permite-nos injetar código no elemento <head> das páginas do painel administrativo.
  6. wp_footer nos permite executar código dentro da função wp_footer().
  7. wp_enqueue_scripts é a ação que registra corretamente os scripts e estilos para o frontend.
  8. admin_notices nos permite mostrar avisos no backend.
  9. widgets_init inicializa os widgets do WordPress e nos permite modificá-los.
  10. delete_user é invocado logo após um usuário ser removido da base de dados.

Parte Três: 50 Ações Para WordPress – 50 Exemplos (11 a 20)

  1. wp_default_styles controla o processo de carregamento dos estilos padrão do WordPress.
  2. get_footer permite-nos alterar a função get_footer().
  3. admin_init é executada toda vez que uma página administrativa é carregada, dando-nos a oportunidade de sermos criativo ao usá-la!
  4. wp_authenticate é executada para autenticar um usuário quando eles realizam o processo de login.
  5. login_form lida com o formulário de login, dando-nos a chance de customizá-lo.
  6. admin_menu permite-nos adicionar/remover itens de menu (ou de sub-menus) do menu principal da administração.
  7. wp, a ação com o menor nome, executada após uma consulta ser analisada, mas logo antes qualquer modelo ser executado.
  8. admin_head-(nome_da_pagina) permite-nos controlar o elemento <head> da página administrativa especificada no nome da ação (nome_da_pagina).
  9. wp_before_admin_bar_render, como o nome sugere, é executada antes da Barra de Ferramenta (anteriormente conhecida como Barra Administrativa) ser renderezida.
  10. profile_update é invocada logo após um perfil de usuário ser atualizado na base de dados.

Parte Quatro: 50 Ações Para WordPress – 50 Exemplos (21 a 30)

  1. pre_get_posts é executada antes da função get_posts(), permitindo-nos brincar com a consulta.
  2. transition_post_status é a ação que controla a transição dos estados das publicações, como "de rascunho para publicado", "agendado para privado" e coisas do tipo.
  3. admin_enqueue_scripts é usada para registrar corretamente scripts e estilos no painel administrativo.
  4. save_post é executa após uma postagem ser salva na base de dados.
  5. add_meta_boxes_(tipo_da_postagem) permite-nos adicionar meta campos a um tipo de postagem customizado na tela de edição de publicação.
  6. activity_box_end é executada ao final da seção "Agora" section.
  7. wp_meta é a ação que nos permite modificar o widget "Meta" padrão.
  8. wp_dashboard_setup inicialização o painel, a página inicial painel administrativo, e nos permite manipulá-lo.
  9. set_current_user é parte da função plugável wp_set_current_user, que altera o usuário atual através do Id ou nome.
  10. plugins_loaded é executada após o carregamento dos arquivos de todos os plugins ativos.

Parte cinco: 50 Ações Para WordPress – 50 Exemplos (31 a 40)

  1. wp_default_scripts lida com a execução dos scripts padrão, permitindo-nos alterá-los no processo.
  2. wp_head nos possibilita executar código na função wp_head().
  3. after_setup_theme é invocada toda vez que um arquivo do tema ativo é carregado.
  4. manage_media_custom_column nos permite adicionar colunas extras à listagem de arquivos na Biblioteca de Mídia.
  5. comment_(estado_antigo)_to_(estado_novo), que possui duas variáveis em seu nome, é executada após o estado do comentário é alterado na base de dados.
  6. template_redirect é invocada quando um "modelo de página" é carregado.
  7. do_feed lida com os feeds da sua instalação WordPress.
  8. admin_bar_menu nos permite manipular a Barra de Ferramentas.
  9. wp_list_categories nos permite controlar o comportamento do widget "Categorias" padrão.
  10. pre_get_search_form é executada antes do formulário de busca do seu tema ser carregado.

Parte Seis: 50 Ações Para WordPress – 50 Exemplos (41 a 50)

  1. admin_head-(pagina_do_plugin) é executada no elemento <head> da página do plugin especificado.
  2. pre_ping é invocada antes de um ping ser processado.
  3. get_header nos permite executar código na função get_header().
  4. login_head é executada na função <head> da página de login e nos permite controlá-la.
  5. admin_footer é executada antes do tag </body> das páginas administrativas.
  6. login_enqueue_scripts é usada para registrar corretamente scripts e estilos na página de login.
  7. manage_users_custom_column permite-nos adicionar colunas customizadas à tabela de Usuários da página "Todos Usuários".
  8. activated_plugin é executada toda vez que um plugin é ativado.
  9. admin_color_scheme_picker controla o comportamento do seletor de "paleta de cores" nas páginas de edição de perfil dos usuários.
  10. wp_logout é executada quando um usuário sai (faz logout) do painel administrativo.
