Programação Orientada a Objetos no WordPress: Introdução

Uma dos motivos para o pessoal amar (ou odiar, mas isso é assunto para outra série) o WordPress é a necessidade de um baixo nível de conhecimento para começar a usá-lo. 

Isso quer dizer que, para o programador mediano, é bem fácil escolhê-lo e começar a criar projetos o utilizando, especialmente se você for acostumado com desenvolvimento front-end.

Claro, se estiver interessado em extender o WordPress através de plugins, widgets ou mesmo através da construção de uma aplicação web baseada em WordPress, a curva de aprendizado é um pouco mais acentuada. Na verdade, para construir widgets, você, geralmente, tem de lançar mão de programação orientada a objetos, se quiser seguir o modelo provido pelo Codex. Além do mais, tem se tornado bem comum ver o paradigma de orientação a objetos sendo usado para plugins famoso, também.

Esse paradigma - embora não seja a solução perfeita como alguns tendem a acreditar - é bem popular por inúmeras razões, mas, antes de falarmos sobre isso, é importante entender o propósito desta série e o que veremos durante as próximas semanas.

"Por Onde Começo?"

Para qualquer pessoa minimamente interessada em aprender uma nova linugagem, ferramenta ou biblioteca, é relativamente fácil começar, especialmente se tiver algum conhecimento em programação.

Mas, aqueles que estão há muito tempo nessa área, costumam esquecer como era passar por momentos que se tentava descobrir o que um código fazia; entender o porque de algo ter sido escrito da forma como o foi; como o autor sabia o que e onde usar; e como descobrir os motivos de certas escolhas para implementação.

Todos passamos por isso um dia, não é? Já olhamos para códigos tentando descobrir como que ele funcionava, até que chegava um hora que perguntávamos "Por Onde Começo?"

A razão dessa série é a de responder essa pergunta.

Orientação a Objetos para Iniciantes

Em específico, essa série focará, somente, naqueles que querem aprender a criar soluções mais complexas com WordPress. Talvez você já se sinta confortável criando temas ou até alguns plugins, mas quer aumentar suas habilidades, aprendendo orientação a objetos.

Claro, o desafio é aprender por onde começar. Óbvio, há inúmeros materiais disponíveis pela web e em livros, tratando dos conceitos básicos de orientação a objetos, mas, acredito ser mais fácil aprender um novo paradigma dentro do contexto de uma base ou bilioteca de verdade.

Assim, os próximos artigos tratarão do básico da orientação a objetos, dentro do ecosistema do WordPress.

Falaremos de vários tópicos, entre eles:

  • classes
  • atributos
  • construtores
  • funções
  • escopo
  • interfaces
  • classes abstratar
  • funções abstratas
  • ...e muito mais.

Ao final da série, você estará bem preparado para, no mínimo, ser capaz de entender códigos, além de ter uma referência sólida, onde você poderá recapitular de tempos em tempos ao usar o WordPress.

O Que Vem A Seguir?

Nosso objetivo não é só fazer você entender como programação orientada a objetos funciona, mas, sim, trabalhar em um exemplo prático na criação de um plugin. Assim, enquanto cobrimos o básico da orientação a objetos, dentro do ambiente do WordPress, também trabalharemos na construção de um plugin para demonstrar as características do paradigma.

Tendo dito, estamos prontos para começar. 

No próximo artigo, começaremos com o bloco fundamental da orientação a objetos: Classes.
