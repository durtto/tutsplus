O Que são Tipos?

Terminamos o artigo anterior com a seguinte afirmação:

Chegaremos lá, porém, precisamos, primeiro, discutir os aspectos primitivos da programação, como as cadeias de caracteres (strings), vetores (arrays) e as estruturas de decisão (condicionais).

Cadeias de caracteres, vetores e outros, são o que podemos chamar de tipos de dados. Repetições e condicionais são que chamamos de estruturas de controle, as quais cobriremos no próximo artigo.

Enquanto isso, aprendamos, de forma simples, o que um tipo de dado representa, para que possamos criar os modelos conceituais em nossa mente, possamos continuar, não só com esse artigo, mas com o resto do conteúdo da série.

Para aqueles que já criaram qualquer código que inclua alguma variável, já devem ter visto algo do tipo:

  • $name = 'Tuts+ WordPress';
  • $is_active = true;
  • $age = 42;
  • ...e por aí vai.

O que mais você ouvirá sobre esse código é que ele trata da definição de variáveis. Isso está correto, mas é uma definição muito generalizada.

Veja, toda vez que você define uma variável em PHP, você o faz prefixando o nome da variável com $. Isso permite o PHP saber que uma variável foi definida e que ela representará algum valor típico.

Bem fácil, certo?

O ponto é, há vários tipos no PHP. Se você é acostumado com "linguagens fortemente tipadas", falaremos sobre isso logo-logo. Porém, saiba que o PHP oferece os seguintes tipos de dados:

  • booleanos
  • inteiros
  • números de ponto flutuantes
  • cadeias de caracteres (strings)
  • vetores (arrays)
  • objetos
  • ...e mais

Booleanos, cadeias de caracteres, números de ponto flutuante e inteiros podem ser vistos como tipos de dados simples, enquanto vetores e objetos podem ser considerados tipos de dados complexoss.

Tipos de Dados Simples

Tipos de dados simples são definidos assim porque os tipos de dados que eles representam são, digamos, simples. Isso significa que eles serão representações de verdadeiro ou falso, decimais, palavras e/ou sentenças.

Para exemplificar, concretamente, cada um dos tipos acima:

  • Valores booleanos são responsáveis por representar valores true (verdadeiro) ou false (falso).
  • Inteiros referem-se a qualquer número completo. Ou seja, números sem partes fracionárias ou casas decimais. Exemplos: -12, -2, 1, 100, 5000, e por aí vai.
  • Números de ponto flutuantes são uma espécie de oposto dos Inteiros, uma vez que eles representarão números com casas decimais e partes fracionárias. Assim, os números 3.1459, 2.5, 100.25 são exemplos válidos. Se você quiser salvar o valor de, digamos, 3/4, você irá, na verdade, salvar o valor .75, a menos que o tenha salvo como uma cadeia de caracteres. Para tanto, você deverá salvar o valor '3/4'. Perceba a sutil diferença entre os dois. Um está entre aspas enquanto o outro não.
  • Por falar em cadeias de caracteres, elas representam uma única palavra ou um conjunto de caracteres alfanuméricos. Palavras, frases ou caracteres aleatórios são representações das cadeias de caracteres. Exemplos concretos incluem 'frases como essa' ou uma única 'palavra' ou algo mais complexo, como 'e952098vjdkw94kd'. 

Atenção–qualquer um dos tipos de dados mencionados acima pode ser transformado em uma cadeia de caracteres envolvendo-os em aspas.

Tem Um Porém

Por exemplo, se você tivesse de trabalhar com o valor booleano truee o guardasse em uma variável, assim: $is_active = 'true';, você apenas teria criado uma cadeia de caracteres, que tem como valor a palavra true nela.

De forma análoga, se criasse variável dessa forma $age = '42';, você só teria criado uma cadeia de caracteres, com 2 caracteres, o 4 seguido do 2, não um número inteiro de valor 42.

Conseguiu entender?

Isso é importante porque, quando começarmos a trabalhar com estruturas de controle–as quais falaremos no próximo artigo–e executarmos comparações, pode ser que obtenhamos algo que não queiramos, caso não soubermos, exatamente, com quais tipos de dados estamos trabalhando.

Tipos de Dados Complexos

Além dos tipos de dados simples, também temos tipos de dados complexos, que, na minha opinião, não são tão complicados. Na verdade, vejo-os como uma forma diferenciada de guardar vários pedaços de informação em uma única variável, enquanto que, com os tipos simples, podemos guardar pedaços únicos de informação em uma única variável.

Bem fácil de lembrar, não é? E quais são os tipos de dados mais complexos?

Os dois principais, os quais focaremos, são os vetores e os objetos. Há inúmeros outros, que estão além do escopo dessa série, então, caso esteja interessado, sinta-se livre para ir ao manual do PHP, mas, aviso: se você for um verdadeiro iniciante, o conteúdo pode aparentar ser bem complicado.

  • Vetores são uma forma de guardar múltiplas partes de uma informação em um conjunto. Imagine, por um momento, que você tem uma lista de nomes que deseja guardar em um conjunto, chamado de $nomes. No PHP, você definirá um vetor e atribuirá a uma variável, dessa forma: $nomes = [ 'John', 'Paul', 'Ringo', 'George' ]; Perceba que temos um vetor de cadeia de caracteres, mas, outro aspecto dos vetores no PHP é que eles podem guardar vários tipos de dados; Isso significa que você pode ter um vetor assim $vetor_com_varios_tipos = [ 42, TRUE, 'Star Wars', .007 ];. Bem legal, não é?
  • Vetores associativos são como os vetores normais que acabamos de ver, exceto que eles são constituidos de pares de chave-valor. Isso significa que qualquer valor é obtido através de uma chave, então, se fôssemos transformar o primeiro vetor em um vetor associativo, ele ficaria mais ou menos assim: array( 'guitarra-ritimica' => 'John', 'baixo' => 'Paul', 'guitarra-solo' => 'George', 'baterias' => 'Ringo' ); Se o formato parecer um pouco estranho, não se preocupe! Veremos um pouco mais sobre o formato mais para a frente nessa série. Mas, o padrão, é que os vetores normais sejam indexados numericamente - isso é, com 0, 1, 2, 3, e por aí vai - e os vetores associativos sejam indexados pelas suas chaves, como 'guitarra-ritmica', 'bateria', e coisas do tipo.
  • Os Objetos, que falamos da última vez, são, sem sombra de dúvidas, os tipos mais complexos de dados que o PHP oferece, porque eles podem ser a combinação de tudo. Seus atributos são compostos de qualquer tipo de dados, inclusive outros objetos e funções, além de poderem realizar operações, não só em tipos básicos de dados, mas em dados complexos também.

Em se tratando de objetos, a melhor maneira de se entender o quão complexos eles são, é vê-los em ação, que é o que faremos quando estivermos construindo um plugin.

Uma Observação Sobre Vetores

Atente que, em algumas linguagens, o tamanho de um vetor deve ser declarado antes dele ser usado. Por exemplo, em C, você tem de dizer ao compilador que o vetor terá, exatamente, 10 posições para guardar valores, por exemplo.

No caso do PHP, isso não se aplica. Os vetores do PHP são dinâmicos, o que significa que você pode continuar adicionando dados ao vetor sem precisar aumentar seu tamanho antes. Sim, como com qualquer coisa, há limites, mas está relacionado ao tamanho dos dados a serem guardados, ou o tanto de memória que a máquina tem disponível.

Não Deixe as Frases Intimidarem Você!

Por hora, não deixe as palavras simples e complexo intimidarem você. Os tipos "simples" são, relativamente, diretos, certo? Mas, "complexo" tem sua conotação, indicando que será complicado, difícil, e isso não é bem verdade - espero que esse artigo tenha deixado isso claro.

Tipos de dados complexos, simplesmente, guardam múltiplos tipos de dados em uma única variável. Tipos de dados simples não o podem.

Existem Outros?

Sim. são eles: resources, callbacks, e pseudo-tipos. A maioria deles vão além do escopo dessa série em particular; porém, podemos até acabar usando as callbacks (que são um tipo particular de função), quando tivermos construindo nosso plugin.

Sinta-se livra para ler o máximo que puder, mas lembre-se que estão além do escopo do conteúdo que cobriremos nessa série. Assim, se você se sentir sobrecarregado, talvez seja melhor revisitá-los após finalizar essa série.

Um Pouco sobre Linguagens Fortemente Tipadas

Algumas linguagens de programação são conhecidas como fracamente tipadas (ou dinâmicas), enquanto outras são conhecidas como fortemente tipadas.

Esse é um outro caso onde a terminologia, geralmente, intimida novatos, ao invés de impulsioná-los a entender e a saber como usar as ferramentas disponíveis. Contudo, a distinção entre as duas é bem simples:

  • Linguagens fortemente tipadas são aquelas em que as variáveis devem ser declaradas com um tipo de dado específico e, além disso, só podem guardar valores daquele tipo de dado. Por exemplo, se quisermos guardar uma cadeia de caracteres, teríamos de declarar a variável como string sMyName; Então, guardaríamos uma cadeia de caracteres nessa variável: sMyName = 'Tom McFarlin';. Isso significa que a variável sMyName só pode guardar cadeias de caracteres. Se tentar guardar outros tipos de dados na variável, o compilador reconhecerá isso como um erro.
  • Linguagens fracamente tipadas (ou dinâmicas), como o PHP, permitem que uma variável possa guardar diversos tipos de dados a qualquer momento da execução. Usando o exemplo acima, poderíamos, facilmente, criar a variável $my_name = 'Tom McFarlin'; e depois atribuí-la outra valor, assim $my_name = false. Ao tentar guardar outro valor na variável, simplesmente, funcionará. Perceba que isso tem suas vantagens e desvantagens.
