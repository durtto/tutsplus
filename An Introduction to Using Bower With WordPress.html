Introdução ao Uso do Bower com WordPress

A maioria dos projetos para web, hoje, começam ou incluem bibliotecas de terceiros. Se você está trabalhando em um tema do WordPress, é bem provável que você esteja usando o HTML5 Boilerplate, Bootstrap, ou o Foundation. A maioria desses projetos estão disponíveis no Github e acessíveis através do Bower.

Nessa série, apresentarei o Bower, falarei sobre como ele pode aprimorar o seu desenvolvimento de temas e plugins para WordPress, além de mostrar como prepará-lo para seu projeto.

O Que é o Bower?

O que, exatamente, é o Bower? De acordo com o site do projeto:

Bower é um gerenciador de pacotes para a web. Oferecendo uma solução genérica e imparcial para o problema de gerenciamento de pacotes front-end, expondo o modelo de dependência de pacote através de uma API que pode ser consumida por um processo de implementações mais dogmático.

Essencialmente, o que isso significa é que o Bower fornece a você uma maneira de especificar quais bibliotecas (ou pacotes) que seu projeto precisa para ser desenvolvido. Ele também é capaz de lidar com as dependências dos projetos que você especificar, de forma automática.

O Bower depende do Node.js e do NPM para funcionar, assim, você precisa ter certeza que os tem instalados na sua máquina. Outro requerimento para trabalhar com o Bower é que os repositórios precisam estar no GitHub e serem adicionados ao Catálogo do Bower.

Bibliotecas de Terceiros

Como mencionem antes, há uma grande chance que você esteja usando uma biblioteca de terceiro em seu projeto. Você provavelmente já foi ao GitHub, baixou o projeto e o colocou os arquivos em um local apropriado em seu projeto. Você precisaria repetir essa tarefa para toda e cada biblioteca que seu projeto precisa.

Com o Bower, você pode instalar (ou baixar), facilmente, essas bibliotecas, sem precisar ir ao GitHub. Todas as bibliotecas que você instala com o Bower, serão colocadas em um diretório chamado bower_components, dentro do diretório raiz do seu projeto. Tudo isso pode ser feito pela linha de comando, assim, não há necessidade de ir e vir do navegador o tempo todo.

Managing Dependencies

Another great feature of Bower is that it manages dependencies for you. It looks at all the packages your project needs and the packages they need as well. Each package might require a specific version of another package, like a specific version of jQuery.

So, for example, let's say your project requires jQuery. You also decide to use both Bootstrap and FitVids as well with your project. Bower will look at all of the different packages and all the versions of packages they rely on as well and figure out what is version needed for all. If there are any conflicts, Bower will let you know.

Who wants to keep track of all the required versions of everything anyway?

No Project Bloat

I mentioned before that all of the packages you install will be placed into a bower_components folder. You can access anything in that folder during your local development, but you don't need to add it to your project's repository.

Let's say you want to use Bootstrap for the layout of your theme and you are using Sass to compile your stylesheets into the style.css file. You can use the bootstrap-sass-official project and simply import the bootstrap.scss file into your existing project .scss files.

I will go into more detail around how to do this in another post.

Conclusion

I talked about what Bower is and how it helps you work with third-party libraries in your project. I also talked about how it can help you manage your dependencies and remove some bloat to your project's repository.
